# 健身计划数据结构说明

## 目标

把外部整理来的健身计划拆成稳定的数据结构，方便后续在小程序中执行、记录和复盘。

当前 `计划.txt` 只作为第一份种子计划来源，不作为长期维护格式。后续新增计划时，都应进入结构化数据，而不是继续堆文本。

## 核心原则

- 官方计划是模板，用户启用后生成自己的计划实例。
- 计划模板允许上架、下架和版本更新。
- 历史训练记录不能因为计划模板更新而丢失。
- 动作库独立维护，计划只引用动作。
- 动作只停用或下架，不做物理删除。
- 中文内容统一按 UTF-8 保存。

## 计划模板字段

`plan_templates`

| 字段 | 说明 |
| --- | --- |
| `id` | 计划唯一标识 |
| `name` | 计划名称 |
| `source_type` | 来源类型，例如 `self_curated`、`licensed`、`user_custom` |
| `goal` | 训练目标 |
| `level` | 难度 |
| `duration_weeks` | 建议周期 |
| `weekly_frequency` | 每周训练次数 |
| `equipment_tags` | 器械标签 |
| `target_users` | 适合人群 |
| `status` | `draft`、`published`、`archived` |
| `version` | 版本号 |
| `summary` | 计划简介 |
| `notes` | 执行说明 |

## 训练日字段

`plan_days`

| 字段 | 说明 |
| --- | --- |
| `id` | 训练日唯一标识 |
| `plan_id` | 所属计划 |
| `day_index` | 计划内排序 |
| `name` | 训练日名称 |
| `focus` | 训练重点 |
| `target_muscles` | 主要肌群 |

## 训练日动作字段

`plan_day_exercises`

| 字段 | 说明 |
| --- | --- |
| `id` | 编排唯一标识 |
| `plan_day_id` | 所属训练日 |
| `exercise_id` | 引用动作 |
| `order` | 动作顺序 |
| `role` | `main` 主项、`assistance` 辅助项、`isolation` 孤立项 |
| `sets` | 建议组数 |
| `reps` | 次数范围或规则 |
| `rpe` | 建议 RPE |
| `rest_seconds` | 建议休息秒数 |
| `weight_rule` | 重量选择规则 |
| `progression_rule` | 递进规则 |
| `notes` | 执行提示 |

## 当前样例计划的结构化结论

样例计划可拆成一个官方计划模板：

- 计划名称：新手三分化增肌计划
- 训练目标：增肌、力量基础、形体塑造
- 适合人群：新手、初中级训练者
- 每周训练次数：3
- 建议周期：12 周
- 训练日数量：3

训练日：

1. 胸 + 三角肌前束 + 三头
2. 背 + 三角肌后束 + 二头
3. 下肢后侧链

其中原文没有明确每个动作的完整组数、次数和休息时间，因此种子数据中先使用可执行默认值：

- 主项：4 组，8-12 次，RPE 8，休息 150 秒。
- 辅助项：3 组，10-12 次，RPE 8，休息 120 秒。
- 孤立项：3 组，12-15 次，RPE 8-9，休息 90 秒。

训练处方默认值：

- 主项：选择能完成目标次数且保留约 2 次余量的重量；全部组达到上限且 RPE 不高于 8 时，下次小幅加重。
- 辅助项：选择中等重量，稳定完成目标次数；动作不变形时再增加次数或小幅加重。
- 孤立项：使用小重量控制轨迹；优先增加次数和控制感，稳定达到上限后再加重。

这些默认值后续应在后台管理端允许人工调整。
