# 页面流程设计

## 1. 页面列表

小程序第一版页面：

- 首页
- 计划库
- 计划详情
- 自定义计划编辑
- 今日训练
- 训练执行
- 动作详情
- 训练总结
- 数据页
- 体重记录
- 我的

后台管理端第一版页面：

- 后台首页
- 动作管理
- 动作编辑
- 计划管理
- 计划编辑
- 用户数据概览

## 2. 小程序主流程

核心训练闭环：

```text
进入小程序
 -> 首页
 -> 计划库
 -> 计划详情
 -> 启用计划
 -> 首页今日训练
 -> 训练执行
 -> 休息计时
 -> 完成训练
 -> 训练总结
 -> 数据页复盘
```

## 3. 首页

目标：让用户快速进入今天要做的训练。

主要模块：

- 当前计划卡片。
- 今日训练入口。
- 训练进度。
- 最近一次训练。
- 最近体重。

状态：

- 未登录：展示登录入口。
- 未启用计划：展示推荐计划入口。
- 已启用计划：展示今日训练。
- 有未完成训练：优先展示继续训练。

关键操作：

- 进入今日训练。
- 继续未完成训练。
- 跳转计划库。
- 跳转体重记录。

## 4. 计划库

目标：让用户找到适合自己的计划。

筛选项：

- 目标：增肌、减脂、塑形、力量。
- 难度：新手、初中级、进阶。
- 器械：徒手、哑铃、杠铃、器械房。
- 每周次数：2 天、3 天、4 天、5 天。

计划卡片展示：

- 计划名称。
- 适合人群。
- 难度。
- 周期。
- 每周训练次数。
- 器械标签。

关键操作：

- 查看计划详情。
- 清空筛选。

## 5. 计划详情

目标：让用户确认计划是否适合自己。

展示内容：

- 计划简介。
- 适合人群。
- 训练目标。
- 周期和频率。
- 需要器械。
- 训练日列表。
- 每个训练日的动作概览。
- 执行注意事项。

关键操作：

- 启用计划。
- 复制为自定义计划。

启用规则：

- 如果用户没有 active 计划，直接启用。
- 如果已有 active 计划，需要确认是否暂停旧计划。
- 启用后生成 `user_plans` 记录，保存当前计划版本。

## 6. 自定义计划编辑

目标：支持用户把外部看到的计划录入成自己的训练计划。

第一版保持简单：

- 填写计划名称。
- 添加训练日。
- 给训练日添加动作。
- 设置组数。
- 设置次数范围。
- 设置休息时间。

动作添加方式：

- 从动作库搜索选择。
- 没搜到时允许临时输入动作名，但标记为 `unmatched`。

保存规则：

- 至少 1 个训练日。
- 每个训练日至少 1 个动作。
- 组数必须大于 0。
- 休息时间默认 90 秒。

## 7. 今日训练

目标：训练开始前确认本次训练内容。

展示内容：

- 当前计划名称。
- 本次训练日名称。
- 动作列表。
- 预计耗时。
- 主要肌群。

关键操作：

- 开始训练。
- 查看动作详情。
- 更换训练日。

## 8. 训练执行

目标：让用户按动作完成训练，并记录每组表现。

页面结构：

- 当前动作名称。
- 动作目标肌群。
- 动作步骤入口。
- 当前组数进度。
- 重量输入。
- 次数输入。
- 休息时间调整。
- 完成当前组按钮。
- 休息计时器。
- 下一个动作按钮。

状态：

- 未开始。
- 训练中。
- 休息中。
- 当前动作完成。
- 本次训练完成。

关键规则：

- 完成一组后自动启动休息计时。
- 用户可以跳过休息。
- 用户可以暂停训练。
- 中途退出时保存 `in_progress` 状态。
- 再次进入首页时优先提示继续训练。

## 9. 动作详情

目标：帮助用户理解动作和肌群。

展示内容：

- 动作名称。
- 主肌群。
- 辅助肌群。
- 器械。
- 难度。
- 动作步骤。
- 常见错误。
- 安全提示。
- 替代动作。

入口：

- 训练执行页。
- 计划详情页。
- 动作库。

## 10. 训练总结

目标：让用户完成训练后获得反馈。

展示内容：

- 总耗时。
- 完成动作数。
- 完成组数。
- 总训练容量。
- 训练密度。
- 预计热量消耗。
- 体重缺失时的补录入口。
- 本次训练肌群。
- 主观疲劳选择。
- 训练备注。

轻递进提示：

- 达成目标次数：提示下次可小幅加重。
- 未达成目标次数：提示下次维持重量。
- 未记录重量：提示下次补充重量，方便递进。

## 11. 数据页

目标：复盘训练和身体趋势。

展示内容：

- 近 7 天训练次数。
- 近 30 天训练次数。
- 训练历史列表。
- 体重趋势图。
- 常练动作。
- 最近训练容量变化。
- 选中训练的时长、密度和预计消耗。
- 本周预计消耗累计。
- 最近 7 次训练消耗趋势。

第一版图表：

- 体重折线图。
- 训练次数柱状图。
- 预计消耗柱状图。

## 12. 体重记录

目标：低成本记录体重变化。

功能：

- 新增体重。
- 编辑体重。
- 删除体重。
- 查看历史记录。

校验：

- 体重低于 20kg 或高于 300kg 时提示二次确认。
- 同一天重复记录时，默认更新当天记录。

## 13. 我的

目标：管理个人资料和计划状态。

展示内容：

- 用户资料。
- 训练目标。
- 训练经验。
- 器械条件。
- 当前计划。
- 历史计划。

关键操作：

- 修改个人资料。
- 暂停当前计划。
- 查看历史计划。

## 14. 后台首页

目标：快速了解基础运营数据。

展示内容：

- 用户数。
- 已发布计划数。
- 已发布动作数。
- 今日训练次数。
- 常用计划排行。

## 15. 动作管理

功能：

- 搜索动作。
- 按肌群、器械、状态筛选。
- 新增动作。
- 编辑动作。
- 上架动作。
- 下架动作。

动作下架规则：

- 下架后不影响历史训练展示。
- 下架后不允许新计划继续选择。

## 16. 计划管理

功能：

- 搜索计划。
- 按目标、难度、状态筛选。
- 新增计划。
- 编辑计划。
- 新增训练日。
- 编排动作。
- 发布新版本。
- 上架计划。
- 下架计划。

版本规则：

- 编辑已发布计划时，生成草稿版本。
- 发布后版本号加 1。
- 用户启用计划时绑定当时版本。
- 历史训练按快照展示。

## 17. 用户数据概览

第一版只看概览，不做复杂用户运营。

展示内容：

- 用户列表。
- 用户当前计划。
- 最近训练时间。
- 累计训练次数。
- 最近体重记录。

限制：

- 不展示敏感隐私详情。
- 不提供人工干预用户计划功能。
