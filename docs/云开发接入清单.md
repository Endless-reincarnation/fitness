# 云开发接入清单

## 当前策略

- 小程序默认仍走本地 mock 和本地缓存。
- 云开发配置集中在 `miniprogram/config/cloud.js`。
- 没有云环境时不要打开 `enabled`，否则容易误判为线上数据问题。
- 页面不要直接调用 `wx.cloud.database()`，统一从 service 层进入。

## 开启步骤

1. 在微信开发者工具中开通云开发环境。
2. 将云环境 ID 填入 `miniprogram/config/cloud.js` 的 `envId`。
3. 将 `enabled` 改为 `true`。
4. 在云数据库中创建集合。
5. 先导入 `seed/cloud-import/` 下的动作、肌群和计划模板数据。
6. 再把 `exerciseService`、`planService`、`workoutService` 从本地实现逐步替换为云数据库实现。

## 集合清单

- `users`：用户资料、目标、经验、器械条件。
- `exercises`：标准动作库。
- `muscles`：肌群字典。
- `plan_templates`：官方计划主表。
- `plan_days`：计划训练日。
- `plan_day_exercises`：训练日动作编排。
- `user_plans`：用户启用的计划实例。
- `workout_sessions`：一次训练记录。
- `workout_sets`：每组训练记录。
- `body_weights`：体重记录。
- `admin_logs`：后台修改日志。

## 权限建议

- `exercises`、`muscles`、`plan_templates`、`plan_days`、`plan_day_exercises`：小程序端只读。
- `user_plans`、`workout_sessions`、`workout_sets`、`body_weights`：用户只能读写自己的数据。
- `admin_logs`：只允许后台管理端写入和查看。
- 官方计划和动作不做物理删除，使用 `status` 字段上架、下架、停用。

## 注意事项

- 官方计划是模板，用户训练的是 `user_plans` 实例。
- 计划模板更新后不要覆盖历史训练记录。
- 动作下架后历史训练仍要显示当时的动作名称。
- 云端写入训练记录前仍要保留本地草稿，防止训练中断丢数据。
