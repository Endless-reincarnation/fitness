# 数据库设计

## 1. 设计原则

- 使用微信云数据库集合承载业务数据。
- 官方计划作为模板保存，用户启用后生成用户计划实例。
- 计划和动作支持下架，不做物理删除。
- 历史训练记录保存当时的动作快照，避免模板变更影响历史展示。
- 所有时间字段统一使用服务端时间。
- 中文内容统一 UTF-8。

| `gender` | string | 否 | `male`、`female`、`unknown` |
| `height_cm` | number | 否 | 身高 |
| `current_weight_kg` | number | 否 | 当前体重 |
| `target_weight_kg` | number | 否 | 目标体重 |

| `weekly_frequency` | number | 否 | 每周计划训练次数 |

## 18. 关键业务约束

- 同一用户同一时间只能有一个 `active` 官方计划。
- 用户可以同时保留多个历史计划。
- 官方计划下架后，不再展示给新用户，但已启用用户仍可查看历史。
- 动作下架后，不再被新计划选择，但历史训练通过快照展示。
- 体重建议限制在 20kg 到 300kg，超出范围需要二次确认。
- 训练中断时，`workout_sessions.status` 保持为 `in_progress`。
