# 云数据库集合配置

## 必建集合

先创建这些集合：

- `muscles`
- `exercises`
- `exercise_alternatives`
- `plan_templates`
- `plan_template_versions`
- `plan_days`
- `plan_day_exercises`
- `user_plans`
- `custom_plans`
- `workout_sessions`
- `workout_sets`
- `body_weights`
- `feedback_messages`
- `users`
- `admin_logs`

## 种子数据导入

按顺序导入 `seed/cloud-import/`：

1. `muscles.json` -> `muscles`
2. `exercises.json` -> `exercises`
3. `plan_templates.json` -> `plan_templates`
4. `plan_template_versions.json` -> `plan_template_versions`
5. `plan_days.json` -> `plan_days`
6. `plan_day_exercises.json` -> `plan_day_exercises`
7. `exercise_alternatives.json` -> `exercise_alternatives`

`manifest.json` 和 `README.md` 不需要导入。

## 权限建议

官方只读数据：

- `muscles`
- `exercises`
- `exercise_alternatives`
- `plan_templates`
- `plan_template_versions`
- `plan_days`
- `plan_day_exercises`

建议权限：所有用户可读，仅管理员可写。第一版没有后台前，可先在控制台手动维护。

用户私有数据：

- `users`
- `user_plans`
- `custom_plans`
- `workout_sessions`
- `workout_sets`
- `body_weights`
- `feedback_messages`

建议权限：仅创建者可读写。

后台日志：

- `admin_logs`

建议权限：仅管理员可读写。

## 验收方式

- 导入后编译小程序，计划页能看到“新手三分化增肌计划”。
- 动作库能看到 15 个动作。
- 完成一次训练后，`workout_sessions` 新增 1 条记录。
- 同一次训练的组记录会进入 `workout_sets`。
- 记录体重后，`body_weights` 出现当天记录。
- 提交建议留言后，`feedback_messages` 新增 1 条记录。
- 断网或权限错误时，小程序仍保留本地记录，并在恢复后自动重试。
