# 项目开发要点

## 通用约定

- 写代码时同步补充必要中文注释。
- 修改中文文件时使用最小范围补丁，避免整文件重写导致编码问题。
- 中文内容统一按 UTF-8 维护。
- 原始训练资料只作为种子来源，长期数据必须结构化维护。

## 产品定位

- 第一版定位为健身计划执行助手，不做课程平台和社区。
- 核心闭环是：选择计划、启用计划、今日训练、记录组数、休息计时、训练总结、数据复盘。
- 官方计划是模板，用户执行的是计划实例。
- 动作库、计划库、训练记录、体重记录后续接微信云开发。

## UI 和皮肤

- 默认皮肤使用 `power-yellow` 黑黄力量风。
- 备选皮肤保留 `tech-green` 黑绿科技风。
- 英文只作为小号氛围标签使用，核心标题、按钮和操作文案必须用中文。
- 页面主视觉要贴近 `docs/ui/pencil-yellow/` 的黑黄设计稿。
- 不要把颜色写死到页面组件里，优先使用 `app.wxss` 里的主题变量。
- 新页面必须接入 `applyTheme(this)`，根节点使用 `theme-{{theme}}`。
- 原生 tabBar 不吃 CSS 变量，切换皮肤时必须调用 `syncTabBarTheme(theme)`。
- tabBar 图标资源在 `miniprogram/assets/tabbar/`，新增皮肤时要补对应图标。

## 训练交互

- 训练页不要强制每组都填完整重量、次数、RPE。
- 次数未填时默认取计划次数下限。
- 同一动作下一组默认沿用上一组填写值。
- 切换到新动作时清空输入，避免误带重量。
- “沿用上一组”只允许沿用当前动作的上一组。
- 计划动作需要展示重量规则和递进规则。
- 完成训练后必须推进到下一个训练日，到最后一天后回到第一天。
- 首页允许用户手动切换当前训练日，用于跳练或调整当天安排。
- 训练中必须保存草稿，首页优先显示“继续训练”，训练完成后清除草稿。
- 自定义计划第一版先走本地缓存，支持多个训练日，保持“创建、启用、执行”的最小闭环。
- 自定义计划动作必须引用标准动作库，避免出现无法展示动作详情的自由文本动作。

## 文案规则

- 核心按钮必须中文，例如“开始训练”“完成本组”“沿用上一组”。
- 英文可以保留为装饰标签，例如 `TODAY PROGRAM`、`LIVE SET`、`SESSION REPORT`。
- 动作清单必须保留副标题，展示组数、次数和休息时间。
